# ADR Index Notes

This directory is the canonical Architecture Decision Record (ADR) archive for the frontend.

## Numbering Policy

- ADRs are append-only records and should not be deleted.
- New ADRs should use the next available number.
- Historical numbering mistakes are documented, not rewritten by default.

## Known Numbering Irregularity

There are two ADRs with index `0008`:

- `0008-dark-mode-implementation.md`
- `0008-markdown-renderer-implementation.md`

This is intentional historical preservation. Do not rename historical ADR files solely to fix numbering unless there is an explicit migration decision.

## Canonical Reading Order

Use filename sort order and commit date context where necessary.
For decision lookup, rely on ADR title and topic rather than numeric ID alone when ambiguity exists.
